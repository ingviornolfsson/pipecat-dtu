# pipecat-quickstart — Fork-Specific Notes

This file documents additions made in this fork on top of the upstream pipecat
framework. It supplements the root `CLAUDE.md`, which covers the overall
framework architecture.

## Directory Layout

```
pipecat-quickstart/server/
  bot.py              — baseline bot (upstream quickstart, minimal edits)
  bot_with_log.py     — extended bot with session logging and smart-turn analysis
  plot_session.py     — offline visualiser for JSONL session logs
  logs/               — per-session JSONL files written at runtime
  todo.md             — in-progress design notes and planned work
```

## Key Additions

### `bot_with_log.py`

Extends `bot.py` with two custom classes:

- **`PeriodicSmartTurnAnalyzer`** — subclass of `LocalSmartTurnAnalyzerV3`.
  Runs turn-probability inference at a fixed interval (default 50 ms) during
  active speech, not only on VAD-stop. Accumulates `TurnMetricsData` objects in
  a thread-safe list; call `drain_metrics()` to consume them.

- **`ConversationLogObserver`** — subclass of `BaseObserver`. Writes a
  per-session JSONL file to `logs/`. Captures three event streams:
  - `bot_speech_changed` — binary start/stop
  - `bot_state_changed` — 5-state FSM: `idle`, `listening`, `user_speaking`,
    `processing`, `speaking`
  - `turn_probability` — smart-turn model score, each inference

  Each line: `{"ts_ns": <int>, "event": "<name>", ...}`. The
  `pipeline_started` event also carries `wall_clock_unix_ns` for anchoring.

  Set `ENABLE_LOGGING = False` (top of file) to suppress all logging.

### `plot_session.py`

Standalone CLI script for visualising a JSONL log:

```bash
uv run plot_session.py logs/session_<ts>_<id>.jsonl
uv run plot_session.py logs/session_<ts>_<id>.jsonl --save plot.png
uv run plot_session.py 0 # Plot the most recent session, -1 for second most recent, etc.
```

Produces vertically stacked subplots sharing a seconds-from-start axis.

## Platform Notes

`daily-python` has no Windows wheels. The `daily` extra has been removed from
`pyproject.toml` and the `DailyRunnerArguments` match branch in `bot.py` is
dead code on Windows. The active transport is `SmallWebRTCTransport`.

## Relevant memory files:
- `pipecat-quickstart/docs/docs/pipecat_lifecycle_frames.md` — analysis of frame types and lifecycle patterns in the pipecat framework, with a focus on concurrency and interruption handling. Useful for designing the Tier 1 event logging schema.
- `pipecat-quickstart/docs/todo.md` — in-progress design notes for the logging implementation, including the event schema, frame pairing properties, and FSM derivation rules. This is a working document and may contain incomplete thoughts or plans.
